# Business Flows（V4.2）

更新时间：2026-03-06
范围：用户后台产品管理、公开分享浏览、移动端筛选与分享交互

## 1. 目标业务闭环

蛋龟选育库当前主闭环：

1. 在用户后台完成结构化记录（宠物、系列、繁育状态、图片）
2. 通过公开分享页对外展示（可浏览、可理解、可转化）
3. 通过统一 UI 与交互规则降低维护成本和学习成本

## 2. 核心用户故事

### 2.1 用户用户：在一个页面完成新建与编辑

**触发场景**
- 用户在 `/app/[tenantSlug]/products` 新建或编辑宠物

**成功标准**
- 不跳转独立详情管理页
- 图片上传、主图设置、排序、资料编辑都在抽屉内完成

**主路径**
1. 打开产品列表页
2. 点击“新建产品”或卡片“编辑”
3. 打开统一抽屉（`ProductDrawer`）
4. 在抽屉中完成图片与表单配置
5. 提交后回到列表并看到变更

**回退路径**
- 取消或关闭抽屉：不污染列表状态
- 提交失败：提示错误并保留已输入内容

### 2.2 用户用户：移动端筛选在不同场景下行为一致

**触发场景**
- 用户在移动端产品列表筛选

**成功标准**
- 交互模式明确，不混淆

**主路径**
1. 顶部筛选区可见时：点击“筛选” -> 锚点弹层
2. 下滑后顶部筛选区隐藏、FAB 出现：点击 FAB -> 底部抽屉
3. 点选药丸立即生效；关键词输入延迟同步

**回退路径**
- 点击遮罩或关闭按钮关闭弹层/抽屉
- 点击“清空”恢复默认筛选

### 2.3 访客用户：公开页浏览不被打断

**触发场景**
- 访客打开分享链接

**成功标准**
- 可直接浏览宠物/系列
- 不强制登录

**主路径**
1. 进入 `/public/s/[shareToken]`
2. 浏览宠物与系列
3. 需要转化时进入“我的”页 CTA
4. 登录后回流用户后台

## 4. 业务状态与交互规则

### MUST

- 产品创建与编辑必须走抽屉，不再新增独立管理页。
- 图片相关操作必须与资料编辑处于同一交互容器（抽屉）。
- 移动端筛选 FAB 必须打开底部抽屉。
- 关闭按钮位置统一为右侧，避免不同容器认知不一致。

### SHOULD

- 系列、性别、状态优先使用药丸点选，减少下拉与复选框。
- 列表页与公开页使用相同筛选药丸视觉语言。
- 共享逻辑优先抽到组件目录，不在页面重复实现。
