## Root Cause

### 1) DB migration 未执行（P0 / GAP）

API 日志明确报错：
- `The column source_meta does not exist in the current database.`
- `The column referral_rewards.trigger_key does not exist in the current database.`

影响：
- 自动绑定接口 `POST /referrals/bind-from-attribution` 失败
- 首只产品创建奖励结算失败
- 邀请中心数据加载异常

日志来源：`/tmp/eggturtle-api.log`

### 2) 本地 public origin 不一致（P0 / ENV）

现象：
- 从 `127.0.0.1:30010` 打开的分享页生成的永久链接，以 `127.0.0.1` 为 origin
- 公开页刷新签名又跳到 `localhost:30010`
- 导致同一环境下出现“某个 origin 可打开、另一个 origin 打不开”的体验撕裂

建议：
- 本地统一只使用一个外部 origin（全部 `localhost` 或全部 `127.0.0.1`）
- 同步设置 `NEXT_PUBLIC_PUBLIC_APP_ORIGIN` / `WEB_PUBLIC_BASE_URL` / `PUBLIC_VERIFY_BASE_URL`

### 3) 详情分享链路仍有产品级路由问题（P1 / PRODUCT）

现象：
- `/public/s/:shareToken/products/:id?src=poster` 在本地表现为空白或回到 feed
- 不满足“详情二维码扫码直达详情页”的验收要求

### 修复结论

### 根因补充
- 生产“夫妻图二维码扫不出来”的根因不是域名，而是服务端未安装 `qrcode` 依赖，`generated-media-renderer` 会降级走伪二维码 SVG fallback，产出的码视觉上像二维码，但实际不可识别。
- 详情页“直达后回落到 feed”的根因是分享签名刷新只请求 `/s/:shareToken`，没有保留 `pid`，导致 302 丢失产品上下文。
