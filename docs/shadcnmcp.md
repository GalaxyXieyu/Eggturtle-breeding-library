# Shadcn-MCP 入门教程：提高前端编程速度

## 📖 背景

在 AI 辅助编程的时代，前端开发面临着一个普遍的问题：**AI 生成的前端代码往往不够美观，缺乏统一的设计风格**。虽然 AI 能够快速生成功能代码，但在 UI/UX 设计方面，它往往无法达到专业设计师的水准。

### 痛点分析

1. **AI 写前端代码的困境**
   - AI 生成的前端代码通常比较"朴素"，缺乏现代感
   - 样式不统一，每个组件都需要手动调整
   - 缺乏设计系统，导致整体界面不协调
   - 需要大量时间进行 UI 优化和美化

2. **组件库使用的挑战**
   - 传统组件库需要查阅文档，效率低
   - AI 无法直接访问组件库的详细信息
   - 组件代码分散，难以快速查找和使用
   - 不同组件库的 API 差异大，学习成本高

### Shadcn 的优势

Shadcn UI 是一个革命性的组件库，它的核心特点是：

- **源码可见**：所有组件都是直接复制到你的项目中，而不是通过 npm 安装
- **完全可定制**：你可以直接修改组件源码，完全控制组件的行为和样式
- **设计统一**：基于 Radix UI 和 Tailwind CSS，提供统一的设计语言
- **AI 友好**：组件代码就在项目中，AI 可以直接读取和理解

## 🎯 Shadcn-MCP 是什么？

Shadcn-MCP 是 shadcn/ui 官方提供的 Model Context Protocol (MCP) 服务器，它允许 AI 助手（如 Cursor）直接访问和搜索 shadcn 组件库中的组件信息。

### 工作原理

```
AI 助手 (Cursor) 
    ↓
Shadcn-MCP 服务器
    ↓
组件注册表 (Registry)
    ↓
组件信息 (代码、示例、依赖)
```

通过 MCP 协议，AI 可以：
- 搜索组件库中的组件
- 查看组件的详细信息和代码
- 获取组件的使用示例
- 了解组件的依赖关系

## ⚠️ 当前限制：注册表格式不统一

虽然 registry.directory 上列出了 30+ 个组件库，但**只有少数几个组件库遵循了标准的 shadcn 注册表格式**。

### 为什么大部分组件库无法使用？

1. **格式不统一**：大部分组件库没有实现标准的 shadcn 注册表 JSON 格式
2. **端点路径不同**：有些组件库使用自定义的端点路径，而不是标准的 `/r` 或 `/registry`
3. **数据结构差异**：返回的数据结构不符合 MCP 工具的预期格式

### 解决方案

- **短期**：使用已适配的组件库（见下方列表）
- **长期**：等待更多组件库适配标准格式，或为特定组件库安装专门的 MCP 服务器

## ✅ 当前可用的组件库

经过全面测试，以下 4 个组件库**完全支持** Shadcn-MCP：

### 1. Shadcn UI（官方）
- **端点**: `https://ui.shadcn.com/r`
- **组件数**: 54 个
- **格式**: 数组格式
- **特点**: 官方组件库，最稳定可靠
- **适用场景**: 基础 UI 组件（按钮、表单、对话框等）

### 2. Aceternity UI
- **端点**: `https://ui.aceternity.com/registry`
- **组件数**: 93 个
- **格式**: 对象格式（items）
- **特点**: 专业级 Next.js 组件，动画效果丰富
- **适用场景**: 现代化、动感强的界面

### 3. JollyUI
- **端点**: `https://www.jollyui.dev/r`
- **组件数**: 35 个
- **格式**: 数组格式
- **特点**: 基于 React Aria，可访问性强
- **适用场景**: 需要无障碍支持的组件

### 4. Dice UI
- **端点**: `https://www.diceui.com/r`
- **组件数**: 26 个
- **格式**: 数组格式
- **特点**: 简洁实用，易于定制
- **适用场景**: 快速开发，轻量级项目

## 🚀 快速开始

### 1. 安装 Shadcn-MCP

在 Cursor 的 MCP 配置文件中添加：

```json
{
  "mcpServers": {
    "shadcn": {
      "command": "npx",
      "args": [
        "shadcn@latest",
        "mcp"
      ]
    }
  }
}
```

配置文件位置：`.cursor/mcp.json`

### 2. 配置组件注册表

在项目的 `components.json` 文件中添加注册表：

```json
{
  "$schema": "https://shadcn-vue.com/schema.json",
  "style": "default",
  "typescript": false,
  "registries": {
    "@shadcn": "https://ui.shadcn.com/r",
    "@aceternity": "https://ui.aceternity.com/registry",
    "@jollyui": "https://www.jollyui.dev/r",
    "@diceui": "https://www.diceui.com/r"
  },
  "tailwind": {
    "config": "tailwind.config.js",
    "css": "src/assets/index.css",
    "baseColor": "slate",
    "cssVariables": true,
    "prefix": ""
  },
  "aliases": {
    "utils": "@/lib/utils",
    "components": "@/components",
    "ui": "@/components/ui",
    "lib": "@/lib",
    "composables": "@/composables"
  }
}
```

### 3. 使用 AI 助手搜索组件

现在你可以在 Cursor 中直接询问 AI：

```
帮我搜索一个适合做聊天界面的组件
```

```
查看 @aceternity 注册表中的动画组件
```

```
给我看看 button 组件的代码示例
```

## 💡 最佳实践

### 1. 选择合适的组件库

- **基础功能**：使用 Shadcn UI
- **动画效果**：使用 Aceternity UI
- **无障碍需求**：使用 JollyUI
- **快速开发**：使用 Dice UI

### 2. 利用源码可见的优势

由于 shadcn 组件是直接复制到项目中的，你可以：

- 让 AI 直接读取组件源码
- 根据需求修改组件
- 保持设计风格统一
- 完全控制组件行为

### 3. 组合使用多个组件库

不同组件库有不同的优势，可以组合使用：

```vue
<!-- 使用 Shadcn 的基础组件 -->
<Button>点击</Button>

<!-- 使用 Aceternity 的动画组件 -->
<AnimatedCard>
  <Content />
</AnimatedCard>
```
